# Migration Delivery Methodology

| Phase | Purpose | Key outputs | Exit criteria |
| --- | --- | --- | --- |
| Discover | Build evidence base | Inventory, risks, usage evidence | Inventory accepted |
| Reduce | Remove waste from scope | Retire candidates, anti-waste score | Scope reduction approved |
| Decide | Lock migration decisions | Decision log, fit-gap, architecture | Steering decisions approved |
| Design | Define target solution | Target architecture, mappings, backlog | Design gate passed |
| Build | Implement target | Extensions, integrations, reports | Build gate passed |
| Validate | Prove solution | Tests, migration trials, reconciliation | UAT gate passed |
| Cutover | Go live | Runbook, rollback, go/no-go | Cutover gate passed |
| Stabilize | Hypercare | Triage, support model, exit criteria | Hypercare exit approved |
