# AX Migration Steering Committee

## Purpose

Prepare decision-focused steering material from AX migration evidence. Focus on decisions, escalations, risk posture, budget/timeline impact, gate readiness, tradeoffs, and next executive actions.

## Default Outputs

- Steering committee pack.
- Decision agenda.
- Executive risk and value summary.
- Escalation register.
- Gate readiness recommendation.
- Meeting minutes and action log.

## Workflow

1. Start with the decisions required in this meeting.
2. Summarize status only where it affects decision, risk, cost, timeline, or compliance.
3. Show options and recommendation for each decision.
4. Assign owner and due date to every action.
5. End with go/no-go, conditional approval, or follow-up requirement.

## Use Generated Artifacts

Prefer these analyzer outputs when available:

- `steering-committee-pack.md`
- `persona-ceo-summary.md`
- `persona-cio-architecture-view.md`
- `persona-ciso-security-view.md`
- `persona-project-manager-control-view.md`
- `weekly-status-report.md`
- `raid-log.md`
- `ai-risk-heatmap.md`
- `ai-value-tracker.md`

## Quality Bar

- Do not let steering packs become long status reports.
- Each recommendation needs impact, risk, cost/timeline signal, owner, and decision deadline.
- Use role-specific language: CEO for value/risk, CIO for architecture, CISO for control/evidence, PM for delivery.
