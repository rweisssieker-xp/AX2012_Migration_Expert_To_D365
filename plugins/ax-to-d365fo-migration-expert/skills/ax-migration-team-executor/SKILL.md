# AX Migration Team Executor

## Purpose

Translate AX migration findings into executable work for delivery teams. Focus on concrete tasks, acceptance criteria, dependencies, test evidence, owner suggestions, and next implementation actions.

## Default Outputs

- Team execution pack.
- Role-specific task list.
- Acceptance criteria.
- Definition of done.
- Workshop question list.
- Test and evidence checklist.
- Backlog-ready user stories.

## Workflow

1. Identify the role or workstream: development, functional, data, integration, reporting, testing, security, cutover, or change.
2. Convert each relevant finding into a task with context, action, acceptance criteria, evidence, dependency, and risk.
3. Use D365FO standard and configuration options before recommending custom build.
4. Include test coverage and documentation expectations.
5. Flag items requiring architect, security, business owner, or steering decision.

## Quality Bar

- Make tasks small enough for sprint planning.
- Include acceptance criteria and evidence for every task.
- Separate analysis tasks from build tasks.
- Preserve traceability to source object, risk, decision, or report.
