# AX Migration Regulatory Pack

## Purpose

Apply regulatory focus areas to migration scope, security gates, evidence, data retention, and cutover decisions.

## Reference

Use `config/regulatory-packs.json` for available packs and focus areas.

## Default Checks

- Data retention and archive strategy.
- Sensitive data and data minimization.
- Audit trail and evidence.
- Financial reconciliation and opening balances.
- Security roles and SoD.
- Integration monitoring and replay.
- Gate approval evidence.

## Quality Bar

Treat regulatory guidance as a checklist and evidence model. Verify legal interpretation against official customer/legal guidance before making definitive compliance claims.
