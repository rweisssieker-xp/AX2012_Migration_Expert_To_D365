# Project Onboarding Guide

## Ziel
Neue Projektmitarbeiter sollen innerhalb eines Tages produktiv werden und dieselbe Faktenbasis nutzen.

## Onboarding Ablauf
1. Plugin installieren und `validate` ausfuehren
2. Beispielanalyse ausfuehren
3. Kundeninventory laden
4. Persona-Reports lesen
5. Eigene Rolle und RACI pruefen
6. Erste Aufgaben aus Team Execution Pack uebernehmen

## Rollenstart
| Rolle | Startdokument | Erste Aktion |
| --- | --- | --- |
| Projektleiter | `project-manager-control-view.md` | RAID und Steering Pack pruefen |
| Developer | `team-execution-pack.md` | Code-Backlog priorisieren |
| Functional Consultant | `team-member-task-view.md` | Fit-gap Fragen klaeren |
| Security Lead | `ciso-security-gate-pack.md` | Security Gate vorbereiten |
| Executive | `persona-ceo-summary.md` | Entscheidungen und Risiken pruefen |
